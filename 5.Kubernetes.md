## 1. Core Architecture

| Layer | Components |
|---|---|
| **Control Plane** | `kube-apiserver`, `etcd`, `kube-scheduler`, `kube-controller-manager`, `cloud-controller-manager` |
| **Node** | `kubelet`, `kube-proxy`, container runtime (containerd/CRI-O) |

- **CRI** (Container Runtime Interface)
- **CNI** (Container Network Interface)
- **CSI** (Container Storage Interface)
- `kubectl` — CLI client

---

## 2. Core Objects / Workloads

| Object | Use case |
|---|---|
| **Pod** | Smallest deployable unit |
| **ReplicaSet** | Maintains a stable set of pod replicas |
| **Deployment** | Declarative updates for Pods/ReplicaSets |
| **StatefulSet** | Stable network identity + storage (databases) |
| **DaemonSet** | One pod per node (log/monitoring agents) |
| **Job** | Run-to-completion task |
| **CronJob** | Scheduled Job |
| **Static Pod** | Managed directly by kubelet, not the API server |

## 3. Networking

```yaml
apiVersion: v1
kind: Service
metadata:
  name: web
spec:
  type: ClusterIP
  selector:
    app: web
  ports:
    - port: 80
      targetPort: 8080
```

| Service type | Behavior |
|---|---|
| **ClusterIP** | Internal-only (default) |
| **NodePort** | Exposes on a static port on every node |
| **LoadBalancer** | Provisions an external cloud LB |
| **ExternalName** | DNS CNAME alias |

- **Ingress** / Ingress Controller
- **Gateway API** — newer, gradually replacing Ingress
- Endpoints / EndpointSlices
- `kube-proxy` modes — iptables, IPVS
- **CoreDNS** — service discovery via `service.namespace.svc.cluster.local`
- **NetworkPolicy** — ingress/egress traffic rules ⚠️ (deny-by-default requires an explicit policy)
- CNI plugins: Calico, Flannel, Cilium, Weave

## 4. Storage

| Object | Role |
|---|---|
| **PersistentVolume (PV)** | Cluster-level storage resource |
| **PersistentVolumeClaim (PVC)** | Namespace-level request for storage |
| **StorageClass** | Dynamic provisioning template |

- Access Modes: `ReadWriteOnce`, `ReadOnlyMany`, `ReadWriteMany`, `ReadWriteOncePod`
- **Reclaim Policy** ⚠️ — `Retain`, `Delete`, `Recycle`
- Volume types: `emptyDir`, `hostPath`, `configMap`, `secret`
- CSI drivers (EBS, Azure Disk, GCE PD)
- VolumeSnapshot

## 6. Scheduling

- Node Affinity / Anti-Affinity
- Pod Affinity / Anti-Affinity
- **Taints and Tolerations** ⚠️ — taints repel pods, tolerations allow (not force) scheduling
- `nodeSelector`
- Resource `requests` & `limits`
- Priority Classes / Preemption
- Topology Spread Constraints
- Manual scheduling (`nodeName`)

<details>
<summary>💡 <b>Gotcha: Taints/Tolerations vs Affinity</b></summary>

- **Taints & Tolerations** — node says "don't schedule here unless you tolerate me." Repels pods away from nodes.
- **Node Affinity** — pod says "I prefer/require this kind of node." Attracts pods toward nodes.
- They're complementary, not interchangeable — dedicated nodes usually use both together.

</details>

## 7. Scaling

| Mechanism | Scales |
|---|---|
| **HPA** (Horizontal Pod Autoscaler) | Number of pod replicas |
| **VPA** (Vertical Pod Autoscaler) | CPU/memory requests of pods |

- Manual scaling (`kubectl scale`)
- **Metrics Server** — required for HPA

## 9. Health & Probes

| Probe | Purpose |
|---|---|
| **Liveness** | Restart container if it fails |
| **Readiness** | Remove from Service endpoints if it fails |
| **Startup** | Delays liveness/readiness until slow-starting app is up |

- Probe mechanisms: HTTP GET, TCP Socket, Exec command, gRPC

## 10. Namespaces & Multi-tenancy

- Default namespaces: `default`, `kube-system`, `kube-public`, `kube-node-lease`
- **ResourceQuota** — caps aggregate resource usage per namespace
- **LimitRange** — default/min/max per-pod resource constraints
- Namespace-scoped vs Cluster-scoped resources ⚠️

## 11. RBAC & Security

- **Role** / **ClusterRole**
- **RoleBinding** / **ClusterRoleBinding**
- **ServiceAccount**
- Authentication — certs, tokens, OIDC
- Authorization modes — RBAC, ABAC, Webhook, Node
- **Admission Controllers** — ValidatingAdmissionWebhook, MutatingAdmissionWebhook, PodSecurity

- **SecurityContext** — `runAsUser`, `runAsNonRoot`, `fsGroup`, `capabilities`
- `kubeconfig` — contexts, clusters, users
- API server audit logging
- Image pull secrets

## 19. Resource Management Internals

| QoS Class | Condition |
|---|---|
| **Guaranteed** | requests == limits for all containers |
| **Burstable** | requests < limits |
| **BestEffort** | no requests/limits set |

- OOMKilled behavior
- CPU throttling (CFS quota)
- Eviction policies (node pressure eviction)

## 21. Comparison Points

> Tying back to the rest of the stack:

| Tool | Role in the K8s lifecycle |
|---|---|
| **Docker/containerd** | Runtime layer underneath Kubernetes |
| **Terraform** | Provisions the cluster infra (nodes, VPC, EKS control plane) |
| **Ansible** | Can bootstrap kubeadm clusters, largely superseded by managed K8s + Helm for app-layer config |
| **GitHub Actions + ArgoCD/Flux** | Common CI (build/push image) + CD (GitOps deploy) split |
